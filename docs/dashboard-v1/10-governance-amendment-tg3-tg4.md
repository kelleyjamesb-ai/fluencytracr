# Governance Amendment — TG3 & TG4 Alignment (Dashboard V1)

## Purpose

This amendment clarifies how Dashboard V1 operates within closed governance boundaries.

Governance is closed.
Dashboard V1 introduces no new evaluative capability.

---

## Reaffirmed Constraints

Dashboard V1 must not:

- Produce scores
- Rank workflows
- Compare teams
- Show trend lines
- Show improvement or decline
- Infer learning or maturity
- Enable individual-level evaluation

---

## Allowed Capability (Clarified)

Dashboard V1 may:

- Surface workflow-level visibility
- Surface one dominant working style per workflow
- Show visibility state
- Show observation window
- Show workflow-level coverage counts

This is observability.
It is not performance.

---

## TG3 Protection (Narrative Resistance)

Dashboard V1 must not enable:

- Longitudinal storytelling
- Before/after comparison
- Time-based progression inference
- Improvement claims

Any feature enabling these requires new governance review.

---

## TG4-A Protection (Observability Ceiling)

Executives may see:

- Workflow-level current state
- Aggregated visibility counts

Executives may not see:

- Individual data
- Team comparisons
- Directional metrics

---

## TG4-B Protection (Decision Boundary)

Dashboard V1 supports decisions about:

- Where guardrails are visible
- Where visibility is limited
- Where workflows may need better instrumentation

Dashboard V1 does not support decisions about:

- Performance
- Personnel
- Productivity
- ROI

---

## Enforcement Clause

If any Dashboard V1 implementation:

- Enables ranking
- Enables trend comparison
- Enables evaluative interpretation

CI must fail.
Feature must not ship.

---

## Governance Status

This amendment operates under:

- Signals not facts
- Organizational scope only
- Default suppress
- Fail closed

Any deviation requires a new governance cycle.
