# Phase 3 Decision ID Determinism Contract

## Scope
This contract applies to unresolved-clause decisions persisted through:
- `PATCH /orgs/:orgId/policies/:policyId/mapping/unresolved/:clauseId`

## Deterministic ID Rule
Decision IDs are generated by `buildDeterministicDecisionId(...)` from the immutable tuple:
- `orgId`
- `policyId`
- `mappingId`
- `clauseId`
- `action`
- `rationale` (trimmed)
- `status` (if present)
- `decidedAt` normalized to second precision

Output format:
- `decision-<sha256-prefix>`

## Replay Behavior
If an unresolved-clause decision request is replayed with identical payload values for the same clause:
1. `decided_at` remains unchanged.
2. Deterministic decision ID remains unchanged.
3. ComplianceDecision persistence is idempotent (`ON CONFLICT (id) DO UPDATE`).

## Non-Deterministic Fields (Excluded)
The following do not affect decision ID generation:
- random event IDs
- response timestamps unrelated to `decidedAt`
- ordering metadata outside the tuple above
