# Deprecated: Maturity Concept

## Status

This document is retained only as a compatibility pointer for older links.

New FluencyTracr work must use [Depth](./DEPTH.md) instead of the older
Maturity framing. Depth is the governed V4 concept for aggregate work
integration.

## Why This Was Superseded

The older Maturity framing overlapped with Depth and could be mistaken for a
stage model, capability label, or scoring system. That creates semantic risk
because FluencyTracr is an aggregate-only value-realization evidence layer, not
a maturity scorer.

Depth preserves the useful part of the old concept - verification, workflow
repertoire, reusable patterns, recovery, and judgment behavior - while making
the governance boundary clearer:

- Depth is not skill, maturity, or performance.
- Depth is aggregate evidence that AI use is becoming integrated into real
  workflows.
- Depth must never be used to rank individuals, teams, managers, or
  departments.
- Depth must be computed and surfaced only at governed aggregate levels.
