# Board Snapshot Specification (Dashboard V1)

## Purpose

The board snapshot provides a clear, current-state view of workflow-level AI interaction patterns.

It does not show performance.
It does not show improvement.
It does not show trends.

---

## Snapshot Contents

For each registered workflow:

- Workflow display name
- Current working style (dominant pattern)
- Visibility state
- Observation window reference

---

## Working Style Labels (Executive Language)

Technical → Executive

- Calibrated Fluency → Balanced AI use
- Blind Efficiency → Fast AI use
- Recovery Maturity → Strong recovery behavior
- Friction Loop → High back-and-forth
- Undertrust Avoidance → AI started but not used

Language must remain neutral.
No good/bad framing.

---

## Visibility Labels

- Clear enough to show
- Not enough data yet
- Not shown (safety)

---

## Display Rules

- No ranking
- No sorting by best/worst
- No departmental grouping
- No prior-period comparison
- No progress indicators
- No trend arrows
- No percent change

Workflows may be listed alphabetically only.

---

## Executive Header

Header may include:

- Observation window
- Number of workflows clear enough to show
- Number not enough data yet
- Number not shown (safety)

Percentages are optional but must refer only to workflows, not people.

---

## Governance Boundaries

If any surface enables:

- Ranking
- Trend inference
- Performance evaluation
- Maturity comparison

The feature must be removed or suppressed.

---

## Suppression Handling

If a workflow is suppressed:

- Do not explain internal logic
- Show neutral safety message
- Do not attempt partial clarity
